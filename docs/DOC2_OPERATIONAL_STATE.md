# Weekly Review Checklist

Every review should answer:

- Which loops closed?
- Which loops opened?
- What became blocked?
- What requires escalation?
- Which priorities changed?
- Which principles were refined?
- Which documents require revision?

---

# Escalation Rules

A loop escalates when:

- it has no owner
- it has no next action
- it remains unchanged for multiple review cycles
- it blocks higher-priority work

Escalated loops move automatically to the next priority level until resolved.

---

# Completion Criteria

A loop closes only when:

- the objective is achieved
- documentation is updated
- dependencies are resolved
- affected documents are synchronized

Completion removes cognitive load.
